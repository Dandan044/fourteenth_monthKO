## 处理来自其他智能体的消息

有时你会收到来自同事（如七月）或其他智能体的消息，这类消息的来源会标记为 `inter_session`。

### 识别方式

消息的 `provenance.kind` 为 `"inter_session"`，表示这是来自其他智能体的转发消息。

### 处理规则

| 消息类型 | 处理方式 |
|----------|----------|
| 报告/分析内容（七月的BTC分析等） | **不要回复**，这些是给你参考或转述用的 |
| `NO_REPLY` | **不要回复**，这是静默确认信号 |
| `HEARTBEAT_OK` | **不要回复**，这是心跳确认 |
| `ANNOUNCE_SKIP` | **不要回复**，这是跳过通知的信号 |
| 真正的问题/请求 | 正常回复 |

### 示例

收到七月的分析报告：
```
## ⚠️ 梭哈做空BTC - 风险评估
...
```
→ 这是给你转述给群友的，不要回复 `NO_REPLY`，因为你已经在转述流程中处理过了。

收到 `NO_REPLY`：
→ 这是系统确认信号，**不要再回复**，否则会造成消息循环。

**简单原则**：来自 `inter_session` 的消息，除非明确是问你问题，否则都当作"已处理"的消息，不需要再回复。
